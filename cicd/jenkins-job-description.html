Jira & Bitbucket Performance Reporter

This job generates on-demand performance and analytics reports by collecting data from our Jira and Bitbucket instances.

Common Use Cases:

  • Analyze a team's performance over a specific date range.
  • Generate a report for each team member covering their individual latest completed sprint.
  • Get a list of all contributors to a specific project.

How to Use:

  1. Click "Build with Parameters".
  2. Choose a User Source:
    • To run for a team, set SOURCE_TYPE to group and SOURCE_IDENTIFIER to your Jira group name (e.g., engineering-team).
    • To run for a specific project, set SOURCE_TYPE to project and SOURCE_IDENTIFIER to the project key (e.g., PROJ).
    • To run for specific people, set SOURCE_TYPE to users and SOURCE_IDENTIFIER to a comma-separated list of usernames (e.g., jdoe,asmith).
  3. Choose a Time Frame:
    • For a custom date range, fill in the Start Date and End Date (format: YYYY-MM-DD).
    • (Recommended) To analyze the most recent sprint for each person, check the LATEST_SPRINT box. This will automatically find the correct sprint and its dates for each user.
  4. Choose Output:
    • Set REPORT_FORMAT to excel for the full, detailed report with charts and hyperlinks.
  5. Click "Build".

Accessing the Report:

After the build completes successfully, a link to the report file (e.g., performance_report.xlsx) will appear in the Build Artifacts on the build's main page.
